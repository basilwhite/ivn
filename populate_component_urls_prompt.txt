Overarching Problem:
This script automates the population of authoritative URLs for components listed in an Excel spreadsheet, reducing manual research and ensuring consistency.

Broader Context:
Organizations often maintain spreadsheets of regulatory, statutory, or policy components. Manually finding and entering URLs for these references is time-consuming and error-prone. Automating this process improves efficiency and accuracy.

High-Level Description:
The script reads an Excel file with columns for component name, ID, and URL. It generates authoritative URLs for blank entries using pattern matching and saves the updated data to a new Excel file.

Purpose and Role:
The script exists to automate and standardize the process of populating URLs for regulatory/policy components in spreadsheets.

Input/Output Definitions:
- Input: Excel file (.xlsx) with columns: component_name, component_id, component_url
- Output: Excel file (.xlsx) with populated component_url values
- User is prompted to provide the input file path or use the default in the script directory

Success Criteria and Error Handling:
- Successful if all blank component_url fields are populated where possible
- Handles missing input file by displaying an error and exiting
- Checks for required columns and raises an error if missing

Evaluation of Output:
- Output file should have the same structure as input, with additional URLs populated
- Output file name: ivntest_with_urls_final.xlsx
- Manual spot-checking or automated tests can verify correct URL generation

Correct Model Behavior:
- Prompt user for input file path
- Validate file existence
- Populate URLs using defined patterns
- Save output and report results

Example Scenario:
- User runs script, is prompted for file path, presses Enter to use default, script processes file, populates URLs, and saves output.
