IVN_Normalize_Dataset.py Prompt File

=== PROBLEM THE SCRIPT SOLVES ===

This script provides robust normalization and denormalization of Excel workbooks for the Integrated Value Network (IVN) project. It ensures that data from a 'Dataset' tab is split into normalized 'Components', 'Sources', and 'Alignments' tabs, and can also reconstruct the original 'Dataset' tab from these normalized tabs. The script enforces strict schema and mapping rules, preserves nulls correctly, and provides an interactive menu for user operation.

=== BROADER CONTEXT ===

The IVN project requires consistent, lossless transformation between a denormalized dataset and a normalized relational structure. This is essential for data quality, interoperability, and downstream analytics. The script is used by analysts and data engineers working with policy, regulatory, and technical documentation in Excel format. The data represents relationships between components from different sources, with metadata about linkages, validity, and alignment rationale.

=== HIGH-LEVEL DESCRIPTION ===

The script provides an interactive menu with five options:
1. Normalize the Dataset tab into Components, Sources, Alignments
2. Generate (denormalize) the Dataset tab from the normalized tabs
3. Evaluate output file against prompt specifications
4. Explain what the script does (displays help text)
5. Exit

User workflow:
1. User runs the script
2. Script displays menu
3. User selects option 1, 2, or 3
4. Script prompts user to navigate to an Excel file using numbered navigation
5. User selects file by number
6. For options 1 and 2: Script prompts for explicit "yes" confirmation
7. Script performs normalization, denormalization, or evaluation
8. Script writes output to timestamped Excel file (options 1 and 2) or displays evaluation report (option 3)
9. Script reports success or error

Evaluation process (Option 3):
- Read the selected output file
- Detect if file is normalized output (has Sources, Components, Alignments) or denormalized (has Dataset tab)
- Validate each tab against expected schema (columns, order)
- Check for 'nan' or 'None' strings (should be blank cells)
- Validate component_id format (should contain '::')
- Report issues and warnings with specific details

Normalization process:
- Read Dataset tab from input Excel file
- Validate all required columns are present
- For each row, extract enabling and dependent side data
- Build Sources dictionary (keyed by source_name, containing source_agency and source_id)
- Build Components dictionary (keyed by source_name::component_name, containing component metadata)
- Build Alignments dictionary (keyed by (enabling_component_id, dependent_component_id), containing alignment metadata)
- Merge duplicate records by choosing longer string value for each field
- Preserve all nulls (never write 'nan' or 'None' strings)
- Write three DataFrames to output Excel file with correct column order

Denormalization process:
- Read Sources, Components, Alignments tabs from input Excel file
- Validate all required columns are present in each tab
- Build lookup dictionaries for sources and components by ID
- For each alignment row, lookup enabling and dependent components and sources
- Reconstruct Dataset row with all required columns
- Preserve all nulls (never write 'nan' or 'None' strings)
- Write Dataset DataFrame to output Excel file with correct column order, plus original normalized tabs

Keyword Extraction Algorithm:
The script uses a multi-step text analysis process to extract keywords and phrases:

1. Acronym Extraction (from original text, preserving case):
   - Pattern: \b[A-Z][A-Z0-9]{1,}(?:-[A-Z0-9]+)*\b
   - Excludes common uppercase words: THE, AND, FOR, THIS, THAT, WITH, FROM, WILL, HAVE, ARE, WAS, BEEN, HAS, HAD, ALL, NOT, BUT, CAN
   - Returns up to 5 unique acronyms

2. Capitalized Phrase Extraction (from original text):
   - Pattern for multi-word proper nouns: \b([A-Z][a-z]+(?:\s+(?:[A-Z][a-z]+|[A-Z]+|of|and|the|for))+)\b
   - Pattern for terms with acronyms: ([A-Z][a-z]+(?:\s+[A-Z][a-z]+)+)\s*\([A-Z]{2,}\)
   - Requires at least 2 words with at least 1 non-stopword
   - Returns up to 5 unique phrases

3. Text Cleaning (for word frequency analysis):
   - Convert to lowercase
   - Remove all non-word characters except spaces and hyphens
   - Normalize whitespace

4. Tokenization:
   - Split on whitespace
   - Filter out words with 3 or fewer characters

5. Word Frequency Analysis:
   - Count occurrences of each word (excluding stopwords)
   - Return up to 10 most common words

6. N-gram Analysis (bigrams and trigrams):
   - Extract sequences of 2-3 consecutive words
   - At least one word in each n-gram must NOT be a stopword
   - Only include n-grams appearing more than once
   - Return up to 7 total n-gram phrases
