Generate a Python script ivn_governance_alignment_engine.py that automates the analysis of U.S. Federal governance to:

· Find Alignment Vectors: Identify where delivering Component A from Document X directly progresses Component B from Document Y toward its intended state.
· Discover & Deconstruct: Continuously crawl .gov domains to find new documents and break them into minimal measurable components.
· Generate Actionable Intelligence: Produce cabinet agency-specific recommendations based on alignment patterns and detected gaps.

2. OPERATING PERSONALITY (SOUL.md INFLUENCE)

The script should embody these cognitive parameters:

· Scope-First Architecture: Broad discovery before deep analysis. Design for horizontal scaling.
· Atomic Deconstruction Mindset: Components are "just barely big enough to measure delivery."
· Gap-Sensitive Logic: Missing expected connections are first-class findings, not just absence.
· Pragmatic Output Orientation: Every analysis must conclude with "Therefore, for Agency Z: [action]".

3. REQUIRED CAPABILITIES (SKILL.md IMPLEMENTATION)

MODULE 1: GovDocument Crawler & Ingestion

```python
# PSEUDO-REQUIREMENTS:
# - Respect robots.txt, prioritize .gov domains
# - Focus on known document types: Public Laws, EOs, OMB Memos, Agency Strategic Plans, CFR updates
# - Extract clean text from PDF, HTML, DOCX
# - Maintain provenance metadata: source URL, publication date, issuing authority
# - Implement incremental crawling (detect new/updated documents)
```

MODULE 2: Atomic Componentizer

```python
# PSEUDO-REQUIREMENTS:
# - Break documents into "atomic components" using rule-based + ML approach
# - COMPONENT DEFINITION: "A directive, requirement, or provision that can be independently tracked for delivery"
# - Examples: "Submit report to Congress by Q4 2025", "Establish interagency working group", "Allocate $50M for pilot program"
# - Tag each component with:
#   * type: ['REQUIREMENT', 'DEADLINE', 'AUTHORITY', 'RESOURCE', 'REPORT', 'GOAL']
#   * responsible_agencies: [list extracted from text]
#   * delivery_metrics: [measurable outcomes implied]
#   * source_document: reference
```

MODULE 3: Alignment Vector Analyzer

```python
# PSEUDO-REQUIREMENTS:
# - Compare components across document pairs
# - Identify "PROGRESSES" relationship when:
#   1. Component A creates a resource Component B needs
#   2. Component A establishes authority Component B requires
#   3. Component A fulfills precondition for Component B
#   4. Component A's output is explicitly referenced as input for Component B
# - Score alignment strength (strong/explicit vs. weak/inferred)
# - Generate alignment chains showing multi-step dependencies
```

MODULE 4: Gap Detection & Recommendation Engine

```python
# PSEUDO-REQUIREMENTS:
# - Detect "GAPS" when:
#   1. Alignment chain is broken (missing intermediate component)
#   2. Component has dependencies but no responsible agency assigned
#   3. Deadline exists without clear preceding milestones
#   4. Resource allocation doesn't match requirement scale
# - Generate agency-specific recommendations that:
#   * Are pragmatic and immediately actionable
#   * Include suggested messaging for strategic communication
#   * Prioritize by: leverage (unblocks multiple alignments), urgency, resource efficiency
```

· Privacy/Security: Only public .gov documents, no authentication required
· Resilience: Handle malformed documents gracefully, continue processing

· Maintainability: Modular design with clear interfaces between components
